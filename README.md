# CloudWS — Cloud Workstation OS

**Self-replicating, immutable, HA-ready workstation OS built on Fedora Rawhide bootc.**

GNOME 50 • Flatpak-first • KVM/QEMU GPU Passthrough • Podman • K3s • Pacemaker HA • NVIDIA RTX 4090

---

## One-Line Install

### Linux (any distro with curl + podman/docker)

```bash
curl -fsSL https://raw.githubusercontent.com/Kabuki94/CloudWS-bootc/main/install.sh | bash
```

### Windows (PowerShell as Administrator)

```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force; irm https://raw.githubusercontent.com/Kabuki94/CloudWS-bootc/main/install.ps1 | iex
```

### Direct bare-metal install (from any Linux live USB)

```bash
sudo podman run --rm -it --privileged --pid=host \
  ghcr.io/kabuki94/cloudws-bootc:latest \
  bootc install to-disk /dev/sdX
```

### Pull pre-built image

```bash
podman pull ghcr.io/kabuki94/cloudws-bootc:latest
```

## What Gets Built

| Target | File | Description |
|--------|------|-------------|
| OCI Image | `localhost/cloudws:latest` | Base container image |
| RAW Disk | `cloudws-bootable.raw` | Auto-sized ext4 disk image |
| Hyper-V | `cloudws-hyperv.vhdx` | Dynamic VHDX for Hyper-V |
| WSL2 | `cloudws-wsl.tar` | WSL2 + WSLg import tarball |
| Installer | `cloudws-installer.iso` | Anaconda installer ISO |
| Live USB | `cloudws-live.iso` | Bootable live environment |
| GHCR | `ghcr.io/kabuki94/cloudws-bootc` | Registry image (auto-update) |

## Architecture

```
Fedora Rawhide fc45 │ Kernel 7.0 │ GNOME 50 "Tokyo" │ Wayland-only
├── ComposeFS + XFS (bare-metal) / ext4 (disk images)
├── OSTree (bootc — immutable, atomic upgrades, rollback)
├── Flatpak-first (22 GNOME apps from gnome-nightly)
├── KVM/QEMU/Libvirt + VFIO GPU Passthrough
├── Podman + K3s + Pacemaker/Corosync HA Clustering
├── NVIDIA RTX 4090 (akmod) + AMD RDNA2 iGPU
├── Looking Glass B7 (low-latency GPU passthrough display)
├── cloud-init (autonomous cloud/hypervisor deployment)
└── Self-replication (cloudws-rebuild → push to GHCR)
```

## Self-Update Loop

CloudWS can update itself from any running instance:

```bash
# Pull latest from GHCR
sudo bootc update

# Or switch to a specific version
sudo bootc switch ghcr.io/kabuki94/cloudws-bootc:20260323

# Rollback if something breaks
sudo bootc rollback

# Rebuild locally from embedded sources + push new version
cloudws-rebuild
```

## Clustering (HA)

Two or more CloudWS nodes can form a high-availability cluster:

```bash
# On node 1:
sudo pcs cluster setup cloudws-cluster node1 node2
sudo pcs cluster start --all

# K3s cluster:
sudo systemctl enable --now k3s          # control plane
# On node 2:
curl -sfL https://get.k3s.io | K3S_URL=https://node1:6443 K3S_TOKEN=<token> sh -
```
